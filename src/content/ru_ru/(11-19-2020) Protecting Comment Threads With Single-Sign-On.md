# [postlink]Защита комментариев с помощью единой авторизации (SSO)[/postlink]

{{#unless isPost}}
Не считаете ваши комментарии публичными? Установлена ли у вас SSO для аутентификации? Это может быть для вас.
{{/unless}}

{{#isPost}}

### <i class="circle">!</i> В этой статье содержится техническая терминология

#### Введение

FastComments SSO <a href="/(4-13-2020)-setting-up-sso-with-fastcomments.html" target="_blank">(подробности здесь)</a> предоставляет вашим пользователям возможность
комментировать без необходимости входа на другую платформу.

Однако этого недостаточно для защиты ваших комментариев, поскольку по умолчанию данные комментариев являются общедоступной информацией - любой, кто может просмотреть
страницу, может видеть комментарии. Однако это обновление меняет ситуацию.

#### Настройка без кода

Мы можем предотвратить просмотр и взаимодействие с нашими комментариями, когда SSO настроено, создав <a href="https://fastcomments.com/auth/my-account/customize-widget" target="_blank">правило настройки</a>.

При этом ищите SSO, и вы найдете эту опцию:

<div class="text-center">
    <img src="/images/fc-sso-protection.png" title="Опция конфигурации для защиты комментариев через SSO" alt="Опция конфигурации для защиты комментариев через SSO" class="lozad" />
</div>

Включите ее и сохраните правило настройки.

#### Защита только определенного домена или страницы

Чтобы защитить только определенный домен или страницу, мы просто настроим правило кастомизации для этого.

В верхней части интерфейса настройки мы найдем два поля: Домен и URL ID.

<div class="text-center">
    <img src="/images/fc-sso-protection-by-page.png" title="Опция конфигурации по странице или домену" alt="Опция конфигурации по странице или домену" class="lozad" />
</div>

Чтобы защитить конкретный домен, введите интересующий домен в поле "домен".

Чтобы защитить конкретную страницу, введите URL страницы в поле "URL ID". Если у вас есть кастомная интеграция с FastComments, вы можете ввести здесь тип ID
вместо URL.

#### Защита за пределами чтения

Включение этой опции защитит страницу или домен от комментариев, если пользователь не вошел в систему через SSO. 

#### Подводные камни

Любые пользователи, создавшие комментарии до вашей интеграции SSO, не смогут их видеть, если не вошли в систему через вашу интеграцию SSO.

#### В заключение

Мы надеемся, что вы нашли этот гид полезным и легким для чтения. Не стесняйтесь оставлять комментарии ниже с любыми вопросами.

Cheers!
