### Что нового

Мы только что выпустили версию 5.1 `fastcomments-react-native-sdk`, нашей библиотеки React Native, которая рендерит комментарии и чат с реальными нативными компонентами вместо WebView.

Мы переписали внутренности, изменили весь внешний вид, добавили виджет живого чата, перенесли SDK на веб и обновили его до последней версии React Native и React.

### Почему две библиотеки React Native?

Краткая заметка, потому что мы получаем этот вопрос. Мы предоставляем два варианта:

- `fastcomments-react-native` — это тонкая обертка вокруг нашего веб-виджета, работающего в WebView. Это самый быстрый способ получить все функции сразу, и он автоматически получает исправления веба.
- `fastcomments-react-native-sdk` (эта) рендерит интерфейс с нативными компонентами React Native без webview. Она более гибкая, полностью настраиваемая и ощущается как нативная, потому что и есть нативная.

Для наилучшего опыта мы рекомендуем SDK. Остальная часть этого поста касается изменений в 5.0.

### Новое управление состоянием

Основной движущей силой этого изменения было желание убедиться, что наша библиотека соответствует своему названию и остается Быстрой. У нас были жалобы от нескольких клиентов на производительность, поэтому теперь это исправлено.

Сначала SDK хранила свое дерево комментариев в Hookstate. Это работало, но по мере увеличения потоков и живых обновлений дело стало замедляться.

Мы заменили Hookstate на Zustand и плоский индексированный хранилище. Теперь комментарии хранятся в `byId` карте наряду с индексами `childrenByParent`, `rootOrder` и `pinnedIds`, вместо дерева, вложенного в состояние.

Плюсы:

- Живые события (новый комментарий, голосование, редактирование) стали O(1) мутациями вместо перерисовки и клонирования дерева.
- Мы отказались от двух полных JSON глубоких клонов, которые запускались на каждом запросе.
- Компоненты подписываются ровно на те фрагменты, которые они читают, стандартная модель селекторов, поэтому голосование по одному комментарию затрагивает только один комментарий.

Этот последний момент важнее, чем может показаться. С подписками на основе селекторов ряд перерисовывается только тогда, когда меняются его собственные данные.

### Переработка на основе токенов

Старый вид представлял собой набор жестко закодированных стилей. Новый стандарт генерируется из набора семантических дизайн токенов (`FastCommentsTheme`): цветов, отступов, радиусов, размеров шрифта, толщины шрифта и размеров аватаров. Вся дерево стилей выводится из этих токенов.

Это означает, что изменение стиля — это всего одна пропс:

```tsx
<FastCommentsLiveCommenting config={config} theme={{ colors: { primary: '#FF5500' } }}/>
```

Темный режим — это всего лишь набор токенов:

```tsx
import { getDarkTheme } from 'fastcomments-react-native-sdk';

<FastCommentsLiveCommenting config={config} theme={getDarkTheme()}/>
```

Сам редизайн представляет собой более чистый и современный внешний вид: тонкие разделители, кнопки и чипсы для голосования, заполненные основные кнопки, округлые аватары и согласованная типографика. Пропс `styles` по-прежнему доступен для хирургических переопределений, так что существующие интеграции продолжают работать.

### Специальный виджет живого чата

Мы добавили `FastCommentsLiveChat`, преднастройку чата на том же движке, который дублирует представление чата нашего Android SDK: хронологические сообщения с самыми новыми внизу, композитор под списком, живая шапка с точкой соединения и количеством пользователей, авто-прокрутка, которая приостанавливается, пока вы читаете старые сообщения, и бесконечный архив при прокрутке вверх. Каждая преднастройка может быть переопределена через `config`.

```tsx
<FastCommentsLiveChat config={{ tenantId: 'demo', urlId: 'my-room' }}/>
```

### Теперь и в вебе

Тот же SDK теперь работает на вебе через `react-native-web`. Композитор редактируемого текста (на основе `react-native-enriched`) рендерит одинаково на iOS, Android и в браузере, так что опыт редактирования остается согласованным везде с единственной реализацией. Оверлей, которые в противном случае могли бы быть обрезаны списком комментариев (меню, выбор GIF, список уведомлений), закрепляются под их триггерами в веб-версии.

### Держим текущий уровень с React Native

5.0 построена и протестирована на React Native 0.81 и React 19, и она нацелена на Новую Архитектуру (Fabric), которая требуется для нативного редактора обогащенного текста. Сохранять актуальность здесь не является рутинной работой: редактор, обработка жестов и рендеринг становятся быстрее и точнее по мере развития React Native, и мы предпочитаем получать эти обновления постепенно, чем отставать на несколько лет.

### В заключение

Это переписывание было направлено на обеспечение тому, чтобы React Native SDK имел такой же первоклассный статус, как и наши другие библиотеки: быстрая и предсказуемая модель данных, внешний вид, который вы можете настроить с помощью одного пропса, виджет чата и поддержка веба, все на современном основании React Native.
