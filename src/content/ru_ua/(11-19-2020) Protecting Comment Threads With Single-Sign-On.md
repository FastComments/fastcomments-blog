# [postlink]Защита Комментариев с Использованием Одного Логина[/postlink]

{{#unless isPost}}
Не считаете свои комментарии публичными? У вас настроен SSO для аутентификации? Это может быть для вас.
{{/unless}}

{{#isPost}}

### <i class="circle">!</i> В этой статье есть технические термины

#### Введение

FastComments SSO <a href="/(4-13-2020)-setting-up-sso-with-fastcomments.html" target="_blank">(подробности здесь)</a> предоставляет вашим пользователям способ
комментировать, не входя в другую платформу.

Однако это само по себе не защищает ваши комментарии, так как по умолчанию данные комментариев являются общедоступной информацией - любой, кто может просмотреть
страницу, может увидеть комментарии. Тем не менее, этот релиз меняет это.

#### Настройка без кода

Мы можем предотвратить просмотр и взаимодействие с нашими комментариями, когда настроен SSO, создав <a href="https://fastcomments.com/auth/my-account/customize-widget" target="_blank">правило настройки</a>.

При этом найдите SSO, и вы увидите этот вариант:

<div class="text-center">
    <img src="/images/fc-sso-protection.png" title="Опция настройки для защиты комментариев через SSO" alt="Опция настройки для защиты комментариев через SSO" class="lozad" />
</div>

Включите его и сохраните правило настройки.

#### Защита только определенного домена или страницы

Чтобы защитить только определенный домен или страницу, просто настроим правило настройки для этого.

В верхней части интерфейса настройки мы найдем два поля ввода: Домен и URL ID.

<div class="text-center">
    <img src="/images/fc-sso-protection-by-page.png" title="Опция настройки по странице или домену" alt="Опция настройки по странице или домену" class="lozad" />
</div>

Чтобы защитить конкретный домен, введите интересующий домен в поле "домен".

Чтобы защитить конкретную страницу, введите URL страницы в поле "URL ID". Если у вас есть пользовательская интеграция с FastComments, вы можете ввести тип ID здесь
вместо URL.

#### Защита за пределами чтения

Включение этой опции защитит страницу или домен от комментариев, если пользователь не вошел в систему через SSO.

#### Подводные камни

Любые пользователи, создавшие комментарии до вашей интеграции SSO, не смогут их увидеть, если не войдут в систему через вашу интеграцию SSO.

#### В заключение

Надеемся, что этот гид был полезен и легок для чтения. Не стесняйтесь комментировать ниже с любыми вопросами.
