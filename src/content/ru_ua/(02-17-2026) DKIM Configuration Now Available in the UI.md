# [postlink]Настройка DKIM теперь доступна в UI[/postlink]

{{#unless isPost}}
Теперь FastComments предоставляет интерфейс для настройки DKIM, что упрощает улучшение показателей доставки электронной почты без использования API.
{{/unless}}

{{#isPost}}

### Что нового

Настройка DKIM для вашего собственного домена электронной почты стала проще. Ранее для настройки DKIM требовалось напрямую использовать
<a href="https://docs.fastcomments.com/guide-api.html#domain-config-structure" target="_blank">Domain Configs API</a>. Теперь вы можете
настроить DKIM прямо из интерфейса администратора FastComments.

### Почему важен DKIM

Когда вы настраиваете домен отправителя для писем, которые FastComments отправляет от вашего имени, провайдеры электронной почты должны проверить, что
Эти письма являются законными. DKIM (DomainKeys Identified Mail) является одним из ключевых механизмов аутентификации, который помогает в этом.
Без правильной настройки DKIM письма, отправленные от вашего домена, с большей вероятностью окажутся в папках со спамом.

### Как это настроить

Перейдите на страницу <a href="https://fastcomments.com/auth/my-account/configure-domains" target="_blank">Мои домены</a> и нажмите
"Показать дополнительные" для домена, который вы хотите настроить. Вы увидите параметры DKIM, где сможете ввести имя вашего домена, селектор ключа
и личный ключ.

После сохранения настройка DKIM будет применена ко всем письмам, отправленным с этого домена.

### API по-прежнему работает

Если вы предпочитаете управлять настройкой DKIM программно, то
<a href="https://docs.fastcomments.com/guide-api.html#domain-config-structure" target="_blank">Domain Configs API</a> продолжает работать
как и раньше.

### В заключение

Мы надеемся, что это упростит гарантирование того, что ваши пользователи получают письма, которые FastComments отправляет от вашего имени. Сообщите нам
ниже, если у вас есть какие-либо вопросы.

Cheers!
