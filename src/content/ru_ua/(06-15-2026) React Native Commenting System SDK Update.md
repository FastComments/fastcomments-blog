### Что нового

Мы только что выпустили версию 5.1 `fastcomments-react-native-sdk`, нашей библиотеки React Native, которая рендерит комментарии и чат с использованием настоящих нативных компонентов вместо WebView.

Мы переписали внутреннюю часть, редизайнули весь интерфейс, добавили виджет чата в реальном времени, перенесли SDK на веб и обновили его до последней версии React Native и React.

### Почему две библиотеки React Native?

Небольшое замечание, потому что мы часто получаем этот вопрос. Мы выпускаем два варианта:

- `fastcomments-react-native` — это тонкая оболочка вокруг нашего веб-виджета, работающая в WebView. Это самый быстрый способ получить все функции мгновенно, и он автоматически получает исправления для веба.
- `fastcomments-react-native-sdk` (это) рендерит интерфейс с использованием нативных компонентов React Native без webview. Он более гибкий, полностью настраиваемый по теме и кажется нативным, потому что является нативным.

Для наилучшего опыта мы рекомендуем SDK. Остальная часть этого поста посвящена тому, что изменилось в 5.0.

### Новое управление состоянием

Основной причиной этого изменения является забота о том, чтобы наша библиотека соответствовала нашему имени и оставалась быстрой. У нас было несколько жалоб от клиентов на ее производительность, и теперь это исправлено.

Изначально SDK хранил свое дерево комментариев в Hookstate. Это работало, но с ростом потоков и живых обновлений производительность начала снижаться.

Мы заменили Hookstate на Zustand и плоский индексированный магазин. Теперь комментарии хранятся в отображении `byId` вместе с индексами `childrenByParent`, `rootOrder` и `pinnedIds`, вместо дерева, вложенного в состояние.

Результаты:

- Живые события (новый комментарий, голосование, редактирование) теперь стали мутациями O(1), вместо обхода и клонирования дерева.
- Мы отказались от двух полных глубоких клонов JSON дерева, которые ранее выполнялись при каждом запросе.
- Компоненты подписываются только на те фрагменты, которые они читают, стандартная модель выбора, поэтому голосование по одному комментарию затрагивает только один комментарий.

Этот последний момент имеет большее значение, чем кажется. С подписками на основе селекторов, строка перерисовывается только тогда, когда изменяются ее собственные данные.

### Редизайн на основе токенов

Старый вид представлял собой кучу жестко закодированных стилей. Новый по умолчанию генерируется на основе набора семантических дизайн-токенов (`FastCommentsTheme`): цвета, отступы, радиусы, размеры шрифтов, веса шрифтов и размеры аватарок. Все дерево стилей выводится из этих токенов.

Это означает, что переоформление осуществляется с помощью одного пропса:

```tsx
<FastCommentsLiveCommenting config={config} theme=\{{ colors: { primary: '#FF5500' } }}/>
```

Темная тема — всего лишь набор токенов:

```tsx
import { getDarkTheme } from 'fastcomments-react-native-sdk';

<FastCommentsLiveCommenting config={config} theme={getDarkTheme()}/>
```

Редизайн сам по себе представляет собой более чистый, современный и нейтральный вид: тонкие разделители, кнопки для голосования в форме капсул и чипсы, заполненные основные кнопки, закругленные аватары и единая типографика. Пропс `styles` все еще доступен для точных замен, так что существующие интеграции продолжают работать.

### Выделенный виджет чата в реальном времени

Мы добавили `FastCommentsLiveChat`, пресет чата на основе того же движка, который дублирует вид чата нашего Android SDK: хронологические сообщения с новыми внизу, составитель под списком, живая шапка с точкой подключения и количеством пользователей, авто-прокрутка, которая приостанавливается, пока вы читаете более старые сообщения, и бесконечная история при прокрутке вверх. Каждый пресет можно переопределить через `config`.

```tsx
<FastCommentsLiveChat config=\{{ tenantId: 'demo', urlId: 'my-room' }}/>
```

### Теперь и в вебе

Тот же самый SDK теперь работает в вебе через `react-native-web`. Составитель богатого текста (поддерживаемый `react-native-enriched`) рендерится одинаково на iOS, Android и в браузере, так что опыт редактирования единообразен везде с одной реализацией. Наложения, которые иначе могли бы быть отсечены списком комментариев (меню, выбор GIF, список уведомлений), укреплены под их триггерами в веб-версии.

### Поддержка актуальности с React Native

5.0 построен и протестирован на React Native 0.81 и React 19, и он нацелен на Новую Архитектуру (Fabric), которую требует нативный редактор богатого текста. Поддержка актуальности здесь не является пустой работой: редактор, обработка жестов и отображение становятся быстрее и правильнее с развитием React Native, и мы предпочли бы получать эти обновления постепенно, чем отставать на годы.

### В заключение

Этот переписывание было направлено на то, чтобы предоставить SDK для React Native такой же первоклассный статус, как и у наших других библиотек: быстрая и предсказуемая модель данных, внешний вид, который можно настроить одним пропсом, виджет чата и поддержка веба, всё это на основе современной платформы React Native.
