# [postlink]잠긴 댓글은 이제 실제로 잠겨 있습니다[/postlink]

{{#unless isPost}}
잠긴 댓글에 잠금 아이콘이 드디어 나타나며, 관리자가 잠금을 해제할 때까지 잠긴 댓글은 더 이상 수정되거나 삭제될 수 없습니다.
{{/unless}}

{{#isPost}}

### 새 소식

모더레이터는 오랫동안 댓글을 잠글 수 있었습니다. 댓글 잠금은 새로운 답글을 방지하는데, 이는 스레드가 마무리되었거나 하위 대화가 엉망이 되었을 때 유용합니다.

문제는 "잠김"이 그렇게 큰 의미를 갖지 못했다는 것입니다. 독자들은 댓글이 잠겼다는 시각적인 표시가 없었습니다. 그리고 새로운 답글이 차단되었음에도 불구하고, 원래 작성자나 편집 접근 권한이 있는 누구나 댓글을 수정하거나 삭제할 수 있었습니다. 이는 정말로 잠긴 것이 아닙니다. 이 업데이트는 두 가지 문제를 해결합니다.

### 잠금 아이콘

모더레이터가 댓글을 잠글 때, 댓글의 오른쪽 상단 코너에 작은 자물쇠 아이콘이 나타나며, 핀 아이콘 바로 옆에 위치합니다. 이는 모든 독자에게 보여지며, 모더레이터만을 위한 것이 아닙니다. 댓글 잠금을 해제하면 아이콘이 사라집니다.

이 아이콘은 핀 아이콘과 동일하게 라이트 모드와 다크 모드를 지원합니다.

### 수정 및 삭제가 차단됩니다

잠긴 댓글은 누구도 수정하거나 삭제할 수 없습니다. 관리자와 모더레이터를 포함하여 누구도 마찬가지입니다. 잠긴 댓글을 수정하거나 삭제하려면 먼저 잠금을 해제하고, 변경한 후 다시 잠그면 됩니다.

이 규칙은 댓글에 영향을 미치는 모든 경로에 적용됩니다:

- 사용자-facing 위젯은 잠긴 댓글에서 수정 및 삭제 메뉴 항목을 숨깁니다.
- 공개 API PATCH 및 DELETE 엔드포인트는 대상 댓글이 잠겨 있으면 `locked` 오류를 반환합니다.
- 관리자의 모더레이션 UI는 잠금을 해제하지 않고 삭제를 시도할 경우 잠금 오류를 표시합니다.

### 예외 사항

몇 가지 경로는 의도적으로 잠금을 우회하여 작업을 수행할 수 있습니다:

- 계정 삭제. 사용자가 FastComments 계정을 삭제하면, 댓글은 잠금 상태와 관계없이 정리됩니다.
- 테넌트 / SSO 사용자 정리. 테넌트 수준의 사용자 제거와 동일한 개념입니다.
- 예정된 자동 삭제. 특정 시간에 댓글 삭제를 예약한 경우, 스케줄러는 잠금에 대해 신경 쓸 필요가 없습니다.
- 중복 정리. 중복 제거 도구는 잠긴 중복 항목을 아무런 절차 없이 제거합니다.
- "이 사용자의 모든 댓글 삭제"와 함께하는 모더레이터 금지. 사용자를 금지하고 그들의 기록을 모두 지우는 것은 단일 의도적인 작업이므로 잠금도 해제됩니다.

나머지 모든 것 - 위젯, 모더레이션 UI, 공개 API, 외부 호출자 - 는 잠금을 존중합니다.

### API 오류 형식

잠긴 댓글을 수정하거나 삭제하기 위해 공개 API를 호출하면 다음과 같은 응답이 반환됩니다:

<div class="code">{
    "status": "failed",
    "code": "locked",
    "reason": "댓글이 잠겼습니다.",
    "translatedError": "이 댓글은 잠겨 있으며 수정하거나 삭제할 수 없습니다. 먼저 잠금을 해제하세요."
}</div>

HTTP 상태는 401입니다. `translatedError` 필드는 귀하의 테넌트에서 설정한 로케일과 설정한 모든 사용자 정의 번역을 사용합니다.

### 결론

작은 변화지만 한동안 열려 있었던 간극을 메꿉니다. 이제 잠긴 것은 실제로 잠긴 것을 의미합니다.

감사합니다!
