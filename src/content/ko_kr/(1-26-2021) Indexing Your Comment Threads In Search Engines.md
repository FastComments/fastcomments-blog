# [postlink]검색 엔진에서 댓글 스레드 인덱싱하기[/postlink]

{{#unless isPost}}
FastComments로 사이트를 크롤링할 때 검색 엔진이 보는 정보를 개선했습니다.
{{/unless}}

{{#isPost}}

## 새로운 소식

이전에 FastComments가 설치된 웹사이트를 크롤링할 때 검색 엔진은 첫 페이지(상위 30개) 댓글만 인덱싱하였습니다.

이것은 이전에 검색 엔진이 귀하의 사이트를 방문할 때 귀하가 보던 것과 동일한 정보를 봤기 때문입니다.

이제 검색 엔진이 귀하의 사이트를 방문하면 모든 댓글이 로드됩니다.

## 페이지 순위에 미치는 영향

현대의 검색 엔진은 페이지를 순위 매길 때 페이지의 콘텐츠 관련성, 검색하는 것과의 비교, 페이지 로드 시간 등 여러 가지를 고려합니다.

이번 변경으로 댓글 스레드의 전체 콘텐츠가 인덱싱 가능해져 검색의 관련성이 잠재적으로 증가합니다. 또한 참여도를 보여주는데, 이는 검색 엔진이 순위를 매기는 데 사용할 수 있습니다.

성능의 잠재적 저하가 있을 수 있지만, 이제 더 많은 댓글을 로드하므로, 우리의 테스트에 따르면 스레드에 수백 개, 아니면 수천 개의 댓글이 있어야 FastComments가 느려져 페이지 로드 시간이 순위에 영향을 미치는 경우가 발생합니다.

예를 들어, 100개 이상의 댓글이 있는 우리의 데모 페이지는 여전히 <a href="https://developers.google.com/speed/pagespeed/insights/?url=https%3A%2F%2Ffastcomments.com%2Fdemo" target="_blank">Google의 PageSpeed Insights에서 99/100 점을 받습니다</a>.

## 우리가 수행한 테스트

이제 검색 엔진이 페이지의 전체 댓글 스레드를 볼 수 있도록 보장하였으며, 전체 댓글 스레드를 한 번에 로드하는 성능을 테스트했습니다.

## 기존 고객에 대한 영향

FastComments를 사용 중인 기존 고객인 경우, 변경 사항이 귀하의 계정에 이미 적용되었으며 추가 작업이 필요하지 않습니다. 검색 엔진이 귀하의 페이지를 다시 인덱싱하면 해당 페이지의 모든 댓글이 인덱싱되어야 합니다.

## 결론

이번 업데이트와 문서화가 유용하셨기를 바랍니다. 즐거운 댓글 작성하세요!
