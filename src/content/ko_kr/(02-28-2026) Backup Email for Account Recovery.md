# [postlink]계정 복구를 위한 백업 이메일[/postlink]

{{#unless isPost}}
FastComments는 이제 계정에 인증된 백업 이메일을 추가하는 것을 지원하여 주 이메일을 잃어버릴 경우 접근을 복구할 수 있는 대체 방법을 제공합니다.
{{/unless}}

{{#isPost}}

### 새로운 소식

이제 FastComments 계정에 **백업 이메일**을 추가할 수 있습니다. 주 이메일에 대한 접근을 잃어버릴 경우, 백업 이메일을 사용하여 기존의 로그인 흐름을 통해 계정을 복구할 수 있습니다.

이 기능은 모든 일반(비 SSO) 사용자에게 제공됩니다.

### 작동 방식

1. [계정 세부정보](https://fastcomments.com/auth/my-account/edit-details) 페이지로 이동합니다.
2. 이메일 필드 아래의 **백업 이메일 정의** 링크를 클릭합니다.
3. 백업 이메일 주소를 입력하고 제출합니다.
4. 확인 이메일을 받기 위해 받은 편지함을 확인하고 확인 링크를 클릭합니다.

확인되면 백업 이메일이 저장되고 필요할 때 사용할 수 있도록 준비됩니다. 동일한 페이지에서 언제든지 변경하거나 제거할 수 있습니다.

### 복구 작동 방식

백업 이메일은 모든 기존 복구 경로와 통합됩니다:

- **사용자 이름 잊어버림** - 사용자 이름 잊어버림 페이지에 백업 이메일을 입력하면 해당 주소로 사용자 이름이 전송됩니다.
- **매직 링크 로그인** - 로그인할 때 이메일 주소로 백업 이메일을 입력합니다. 매직 링크가 백업 이메일로 전송됩니다.
- **비밀번호 로그인** - 계정을 찾기 위해 사용자 이름/이메일 필드에 백업 이메일을 입력할 수 있습니다.

새로운 복구 흐름이 필요하지 않았습니다. 백업 이메일은 주 이메일이 인증 목적으로 사용하는 모든 곳에서 작동합니다.

### 무엇이 아닌가

백업 이메일은 **오직** 계정 복구에만 사용됩니다. 알림, 요약 이메일 또는 기타 통신을 받지 않습니다. 이는 안전망이지 두 번째 받은 편지함이 아닙니다.

### 결론

이메일 계정에 대한 접근을 잃는 것은 그와 연결된 모든 서비스에 대한 접근도 잃는 것보다 더 스트레스를 주는 일입니다. 이 작은 추가가 여러분에게 평온함을 제공하기를 바랍니다.
