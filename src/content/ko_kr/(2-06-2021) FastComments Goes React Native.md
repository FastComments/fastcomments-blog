### <i class="circle">!</i> 이 기사에는 기술 용어가 포함되어 있습니다

### 새로운 소식

FastComments는 이제 VanillaJS 위젯과 동등한 기능을 갖춘 React Native 구성 요소를 제공합니다.

저희는 이 라이브러리를 `fastcomments-react-native`라고 부르기로 결정했습니다. 소스 코드를 포함한 GitHub 저장소는 <a href="https://github.com/FastComments/fastcomments-react-native" target="_blank">여기</a>에서 확인할 수 있습니다.

저장소에는 위젯의 사용 예제도 포함되어 있습니다.

NPM에 등록되어 있으며, <a href="https://www.npmjs.com/package/fastcomments-react-native" target="_blank">여기</a>에서 찾을 수 있습니다.

### 작동 방식

이 새로운 구성 요소의 작동 방식은 React 구성 요소가 기존 FastComments 위젯의 래퍼라는 것입니다.

이는 기존 구성 요소에 기능을 추가하거나 버그를 수정하더라도 즉시 혜택을 누릴 수 있다는 의미입니다!

### 개발 시간

React Native 위젯은 소요되는 개발 시간이 거의 없이 몇 분 만에 애플리케이션에 추가할 수 있습니다.

### SSO

모든 React Native 라이브러리는 기본적으로 SSO를 지원하며, SSO 통합 프로세스는 Vanilla JS 구성 요소와 동일합니다. 기존의
백엔드 코드 예제도 적용됩니다.

### 사용자 정의 스타일링?

React Native 구성 요소는 웹 기반 위젯과 동일하게 위젯 사용자 정의 UI를 통해 CSS를 사용하여 스타일을 지정할 수 있습니다.

### 대안

자신만의 UI를 구축할 수 있습니다.

### 결론

`fastcomments-react-native` 라이브러리와 향후 라이브러리를 출시함으로써, 현대 개발 방법론을 사용하면서 FastComments를 채택하는 데 있어 개발자들이 더 쉽게 접근할 수 있기를 바랍니다.

건배!
