# [postlink]FastComments가 이제 Markdown을 지원합니다[/postlink]

Markdown을 사용하여 FastComments에서 더 풍부한 댓글 경험이 제공됩니다!

{{#isPost}}

Markdown은 텍스트 스타일링을 위한 규칙 집합입니다. 예를 들어 굵은 단어를 만들기 위해 \*\*이렇게\*\* 할 수 있습니다.
일반 링크를 삽입하는 것은 원시 URL을 붙여넣는 것으로 충분하며, 이미지 또한 이전과 같이 작동합니다.

다음은 Github에서 제공하는 Markdown 요약표입니다: <a href="https://enterprise.github.com/downloads/en/markdown-cheatsheet.pdf" target="_blank">https://enterprise.github.com/downloads/en/markdown-cheatsheet.pdf</a>.

### FastComments가 지원하는 Markdown 구문은 무엇인가요?

우리는 제목을 제외한 모든 Markdown을 완전히 지원합니다. 제목을 정의할 때는 대화의 흐름을 방해하지 않기 위해 일반 텍스트로 렌더링합니다. 제목은 블로그 게시물이나 문서와 같은 콘텐츠에 적합합니다 - 대화에는 적합하지 않습니다.

FastComments는 항상 [img]...[/img] 구문을 통해 이미지를 지원해왔습니다. 이는 Markdown보다 이해하기 더 쉽기 때문에 계속 지원될 것입니다.

### 콘텐츠 혼합하기

단일 댓글에는 모든 형식의 포맷팅 - markdown, 링크, 이미지, 코드 조각 및 테이블을 포함할 수 있습니다.

### XSF/XSS 공격 방지

이전에는 FastComments가 네트워크를 통해 일반 텍스트를 전송하고 클라이언트에서 파싱했습니다. 이제는 HTML이 네트워크를 통해 전송되며, 댓글을 편집하는 기능과 같은 기능을 지원하기 위해 데이터베이스에 모든 댓글의 두 가지 버전을 유지합니다.

HTML을 댓글에 직접 전송하는 것에는 위험이 따릅니다 - 따라서 생성된 모든 댓글은 크로스 사이트 및 크로스 프레임 공격을 방지하기 위해 파싱되고 청소됩니다.

### 데이터 내보내기

댓글을 내보낼 때 사용자가 댓글을 입력할 때 본 버전을 받게 됩니다. 원시 HTML을 얻고 싶으시다면 저희에게 연락해 주세요.

### 성능 문제

기능 추가는 종종 속도를 늦추는 경향이 있습니다. 우리는 FastComments에서 그런 것을 하고 싶지 않습니다.
이 기능을 추가하면서 클라이언트 측 위젯의 크기가 약 400바이트 줄어들었습니다 - 9.53kb에서 9.12kb로 - 이는 페이지 로드를 더 빠르게 의미합니다.

이론적으로 이 기능이 댓글 저장 메커니즘을 느리게 할 수 있지만, 우리는 성능 저하를 관찰하지 않았으며 항상 API를 모니터링하고 있습니다.

### 어떻게 얻을 수 있나요?

Markdown 지원은 모든 고객에게 자동으로 배포되었습니다. 귀하의 사이트에 게시된 콘텐츠에서 큰 변화는 보이지 않을 것입니다 - 이제 사용자는 굵게, 밑줄을 긋고 간단한 테이블을 생성하는 등의 멋진 작업을 할 수 있습니다.

모든 기능 출시와 마찬가지로 피드백을 기다리며, 많은 자동화된 테스트가 있지만... 문제가 발생하면 알려주세요!
