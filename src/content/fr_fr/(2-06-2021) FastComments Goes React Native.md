# [postlink]FastComments passe à React Native[/postlink]

### Quoi de neuf

FastComments dispose désormais de composants React Native qui sont au même niveau de fonctionnalités que notre widget VanillaJS.

Nous avons décidé d'appeler la bibliothèque `fastcomments-react-native`. Vous pouvez trouver le repository GitHub avec le code source <a href="https://github.com/FastComments/fastcomments-react-native" target="_blank">ici</a>.

Le repository contient également des exemples d'utilisation du widget.

Il est sur NPM, que vous pouvez trouver <a href="https://www.npmjs.com/package/fastcomments-react-native" target="_blank">ici</a>.

### Comment ça marche

Le fonctionnement de ce nouveau composant est que le composant React est un wrapper autour de l'existant widget FastComments.
 
Cela signifie que si nous ajoutons des fonctionnalités à nos composants existants ou corrigeons des bugs, vous en bénéficierez immédiatement !

### Temps de développement

Les widgets React Native peuvent être ajoutés à votre application en quelques minutes avec peu de temps de développement requis.

### SSO

Toutes les bibliothèques React Native prennent en charge le SSO dès la sortie de la boîte, et le processus d'intégration SSO est le même que pour les composants Vanilla JS. Les exemples de code backend existants s'appliquent également.

### Personnalisation des styles ?

Les composants React Native peuvent être stylisés à l'aide de CSS via l'interface UI de personnalisation du widget, de la même manière que les widgets basés sur le web.
