### Τι Νέο

Δημιουργήσαμε την έκδοση 5.1 του `fastcomments-react-native-sdk`, της βιβλιοθήκης React Native που αναπαριστά σχόλια και συνομιλία με πραγματικά εγγενή στοιχεία αντί για WebView.

Επανασχεδιάσαμε τα εσωτερικά, ανασχεδιάσαμε την όλη εμφάνιση, προσθέσαμε ένα widget ζωντανής συνομιλίας, φέραμε το SDK στον ιστό και το αναβαθμίσαμε στην τελευταία έκδοση των React Native και React.

### Γιατί Δύο Βιβλιοθήκες React Native;

Μια γρήγορη σημείωση, γιατί λαμβάνουμε αυτή την ερώτηση. Προσφέρουμε δύο επιλογές:

- `fastcomments-react-native` είναι μια λεπτή επικάλυψη γύρω από το widget ιστού μας που τρέχει σε WebView. Είναι ο ταχύτερος τρόπος για να αποκτήσεις κάθε χαρακτηριστικό άμεσα, και επωφελείται από τις διορθώσεις του ιστού αυτόματα.
- `fastcomments-react-native-sdk` (αυτό) αναπαριστά το UI με εγγενή στοιχεία React Native χωρίς WebView. Είναι πιο ευέλικτο, πλήρως θεματοθετημένο και έχει αίσθηση εγγενών στοιχείων γιατί είναι εγγενές.

Για την καλύτερη εμπειρία προτείνουμε το SDK. Το υπόλοιπο αυτού του άρθρου αφορά τα όσα άλλαξαν στην 5.0.

### Νέα Διαχείριση Κατάστασης

Ο κύριος στόχος αυτής της αλλαγής είναι να διασφαλίσουμε ότι η βιβλιοθήκη μας παραμένει πιστή στο όνομά της και παραμένει Γρήγορη. Είχαμε αρκετούς πελάτες να παραπονούνται για την απόδοσή της, οπότε αυτό τώρα έχει διορθωθεί.

Το SDK αρχικά κρατούσε το δέντρο σχολίων του σε Hookstate. Λειτουργούσε, αλλά καθώς οι νήματα και οι ζωντανές ενημερώσεις αυξάνονταν, τα πράγματα άρχισαν να επιβραδύνονται.

Αντικαταστήσαμε το Hookstate με το Zustand και μια επίπεδη, ευρετηριασμένη αποθήκη. Τα σχόλια τώρα βρίσκονται σε έναν χάρτη `byId` δίπλα σε ευρετηριασμούς `childrenByParent`, `rootOrder` και `pinnedIds`, αντί για ένα δέντρο εντός της κατάστασης.

Το όφελος:

- Οι ζωντανές εκδηλώσεις (ένα νέο σχόλιο, μια ψήφος, μια επεξεργασία) έγιναν μεταβολές O(1) αντί να περπατούν και να κλωνοποιούν ένα δέντρο.
- Αποσύραμε δύο πλήρεις JSON βαθιές κλώνες δέντρου που χρησιμοποιούνταν σε κάθε λήψη.
- Τα στοιχεία εγγράφονται ακριβώς στις φέτες που διαβάζουν, το πρότυπο μοντέλο επιλεγέα, έτσι μια ψήφος σε ένα σχόλιο επηρεάζει μόνο ένα σχόλιο.

Αυτό το τελευταίο σημείο έχει μεγαλύτερη σημασία από ό,τι ακούγεται. Με τις συνδρομές βάσει επιλεγέα, μια σειρά ξανασχεδιάζεται μόνο όταν αλλάζουν τα δικά της δεδομένα.

### Μια Επανασχεδίαση Βασισμένη σε Tokens

Η παλιά εμφάνιση ήταν μια σωρό στυλ που είχαν γίνει σκληρά. Η νέα προεπιλογή παράγεται από ένα σύνολο σημασιολογικών design tokens (`FastCommentsTheme`): χρώματα, αποστάσεις, ακτίνες, μεγέθη γραμματοσειρών, βάρη γραμματοσειρών και μεγέθη avatar. Όλη η ιεραρχία στυλ προέρχεται από αυτά τα tokens.

Αυτό σημαίνει ότι η επανασχεδίαση είναι μια ιδιότητα:

```tsx
<FastCommentsLiveCommenting config={config} theme=\{{ colors: { primary: '#FF5500' } }}/>
```

Η σκοτεινή λειτουργία είναι μόνο ένα σετ tokens μακριά:

```tsx
import { getDarkTheme } from 'fastcomments-react-native-sdk';

<FastCommentsLiveCommenting config={config} theme={getDarkTheme()}/>
```

Η επανασχεδίαση αυτή καθεαυτή είναι μια πιο καθαρή, σύγχρονη-ουδέτερη εμφάνιση: χωρίσματα λεπτού πάχους, κουμπιά ψήφου και chips σε χάπι, γεμάτα κύρια κουμπιά, στρογγυλεμένα avatars, και μια συνεπή κλίμακα τύπου. Η ιδιότητα `styles` παραμένει εκεί για χειρουργικές υπερισχύσεις, οπότε οι υπάρχουσες ενσωματώσεις συνεχίζουν να λειτουργούν.

### Ένα Αφιερωμένο Widget Ζωντανής Συνομιλίας

Προσθέσαμε το `FastCommentsLiveChat`, μια προρυθμισμένη συνομιλία πάνω από την ίδια μηχανή που αντικατοπτρίζει την προβολή συνομιλίας του Android SDK μας: χρονολογικές μηνύματα με τα πιο πρόσφατα στο κάτω μέρος, ο συντάκτης κάτω από τη λίστα, μια ζωντανή ταινία κεφαλίδας με μια ένδειξη σύνδεσης και αριθμό χρηστών, αυτόματη κύλιση που σταματά καθώς διαβάζετε παλαιότερα μηνύματα, και απεριόριστη ιστορία καθώς κυλίετε προς τα πάνω. Κάθε προρύθμιστη είναι υπερκαλύψιμη μέσω `config`.

```tsx
<FastCommentsLiveChat config=\{{ tenantId: 'demo', urlId: 'my-room' }}/>
```

### Τώρα και στον Ιστό

Το ίδιο SDK τώρα τρέχει στον ιστό μέσω του `react-native-web`. Ο πλούσιος συντάκτης κειμένου (powered by `react-native-enriched`) αναπαρίσταται με τον ίδιο τρόπο σε iOS, Android, και τον περιηγητή, έτσι η εμπειρία επεξεργασίας είναι συνεπής παντού με μια μόνο υλοποίηση. Οι επικάλυψεις που διαφορετικά θα κοβόταν από τη λίστα σχολίων (μενού, ο επιλογέας GIF, η λίστα ειδοποιήσεων) είναι αγκυροβολημένες κάτω από τους προκλητές τους στην κατασκευή του ιστού.

### Διατηρώντας Ενημερωμένο με το React Native

Η 5.0 είναι κατασκευασμένη και δοκιμασμένη με το React Native 0.81 και το React 19, και στοχεύει στη Νέα Αρχιτεκτονική (Fabric), την οποία απαιτεί ο εγ native πλούσιος επεξεργαστής κειμένου. Η διατήρηση ενημέρωσης εδώ δεν είναι εργασία χωρίς σημασία: ο επεξεργαστής, η διαχείριση κινήσεων και η αναπαράσταση γίνονται όλοι γρηγορότεροι και πιο σωστοί καθώς το React Native προχωρά, και θα προτιμούσαμε να πάρουμε αυτές τις αναβαθμίσεις σταδιακά παρά να μείνουμε χρόνια πίσω.
