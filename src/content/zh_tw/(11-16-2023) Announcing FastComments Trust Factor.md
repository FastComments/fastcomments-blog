# [postlink]宣布 FastComments 信任因子[/postlink]

{{#unless isPost}}
FastComments 現在在判定什麼是垃圾郵件時，使用了更多的啟發式方法。
{{/unless}}

{{#isPost}}

### 新聞

截至 2023 年 11 月 16 日，FastComments 開始擴展判定用戶可信度的方式，以減少處理垃圾郵件過濾器的挫折感。

### 對用戶的影響

如果您是現有網站的長期成員，而且您的評論曾被垃圾郵件過濾器攔截，這項變更適合您。

我們收到很多報告，指出用戶在網站上執行一些垃圾郵件過濾器不喜歡的操作，例如分享大量鏈接等等，這些評論最終進入垃圾郵件，並根據網站設置被阻止或等待版主批准。

### 對網站管理員和版主的影響

1. 您可能會看到來自您最活躍用戶的評論自動標記為垃圾郵件的次數減少。
2. 這不會改變基於文字/短語黑名單的垃圾郵件檢測。如果您有此設置，粗言穢語和過濾短語仍然會導致評論進入垃圾郵件。

信任因子的數字目前不對網站管理員或版主公開，但我們將在未來探索這一點。

### 信任因子的計算方式

信任因子是一個從 `0` 到 `100` 的數字，並在每個網站基礎上維護。

以下變數用於確定信任因子：

- 與網站的互動年齡。
- 已批准的評論數量。
- 固定評論的數量。

如果您已經是某個網站的成員超過六個月，並且您留下了超過 50 條已批准的評論，您的信任分數將達到完美的 `100`。

否則，公式如下，並將隨著時間的推移而演變：

    min((timeFactor + commentFactor + pinFactor) / 3, 100)

其中：

    timeFactor = 100 * TimeSinceFirstComment / SixMonths
    commentFactor = NumberOfApprovedComments
    pinFactor = NumberOfPinnedComments * 20

您可能會注意到這裡的 `* 20` - 固定評論的權重很高。如果版主將您的評論固定，您可能會被視為相當受信任的用戶。

### 對開發人員的注意事項

在 API 中，`UserBadgeProgress` 現在已公開 `autoTrustFactor` 和 `manualTrustFactor`。

`autoTrustFactor` 是由我們計算的，無法通過 API 寫入。

如果您希望自己配置信任因子，可以定義 `manualTrustFactor`。系統將使用該值，並且我們將繼續單獨維護 `autoTrustFactor` 的值。

### 信任因子的濫用

和所有事情一樣，我們預期這會被濫用。人們會與社區建立關係，然後利用帳戶發佈垃圾郵件。然而，我們認為通過合理的默認設定（六個月，50+ 條評論），我們設定的門檻足夠高，以至於對大多數垃圾郵件發送者來說不值得。在他們開始發佈垃圾郵件後，帳戶可以立即被版主禁用。在未來，我們也可能讓版主調整信任因子。
