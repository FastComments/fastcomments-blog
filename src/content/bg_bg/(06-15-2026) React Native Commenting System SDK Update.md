# [postlink]Актуализация на SDK за коментари на React Native[/postlink]

{{#unless isPost}}

### Какво ново

Току-що пуснахме версия 5.1 на `fastcomments-react-native-sdk`, нашата библиотека за React Native, която рендира коментари и чат с реални оригинални компоненти вместо WebView.

Пренаписахме вътрешностите, редизайнахме целия външен вид, добавихме уиджет за живо чат, донесохме SDK на уеб и го обновихме до най-новите версии на React Native и React.

### Защо две библиотеки за React Native?

Бърза бележка, защото получаваме този въпрос. Предлагаме две възможности:

- `fastcomments-react-native` е тънък обвивка около нашия уеб уиджет, работещ в WebView. Това е най-бързият начин да получите всяка функция моментално и автоматично се възползва от уеб поправки.
- `fastcomments-react-native-sdk` (това) рендира UI с оригинални компоненти на React Native без webview. Той е по-гъвкав, напълно темиран, и изглежда оригинално, защото е оригинално.

За най-добро преживяване препоръчваме SDK. Останалата част от този пост е за това, което се е променило в 5.0.

### Ново управление на състоянието

Основната причина за тази промяна е да се уверим, че нашата библиотека остава вярна на името си и остава бърза. Имахме няколко клиента, които се жалваха за производителността ѝ, така че това сега е поправено.

SDK първоначално държеше дървото от коментари в Hookstate. Работеше, но с растежа на нишките и живите актуализации нещата започнаха да се забавят.

Заменихме Hookstate с Zustand и плосък, индексен магазин. Коментарите сега живеят в `byId` карта заедно с `childrenByParent`, `rootOrder` и `pinnedIds` индекси, вместо в дърво, вложено в състоянието.

Плащането:

- Живите събития (нов коментар, гласуване, редакция) станаха O(1) мутации вместо обикаляне и клониране на дърво.
- Премахнахме две пълни JSON дълбоки клони, които бяха активни при всяко извличане.
- Компонентите се абонират точно за тези срезове, които четат, стандартният модел на избирач, така че гласуването за един коментар засяга само един коментар.

Точката на последното е по-важна, отколкото звучи. С абонаменти, базирани на избирачи, реда се обновява само когато собствените му данни се променят.

### Редизайн на база токени

Старият външен вид беше купчина хардкодирани стилове. Новият по подразбиране се генерира от набор от семантични дизайн токени (`FastCommentsTheme`): цветове, разстояние, радиус, размери на шрифтове, тегла на шрифтове и размери на аватари. Цялото дърво от стилове произлиза от тези токени.

Това означава, че промените на стиловете стават с един пропс:

```tsx
<FastCommentsLiveCommenting config={config} theme={{ colors: { primary: '#FF5500' } }}/>
```

Тъмният режим е на един комплект токени разстояние:

```tsx
import { getDarkTheme } from 'fastcomments-react-native-sdk';

<FastCommentsLiveCommenting config={config} theme={getDarkTheme()}/>
```

Редизайнът сам по себе си представлява по-чист, модерен неутрален вид: тънки разделители, капсулирани бутони за гласуване и чипове, запълнени основни бутони, закръглени аватари и последователна типографска скала. Пропсът `styles` все пак присъства за хирургически замени, така че съществуващите интеграции продължават да работят.

### Специален уиджет за живо чат

Добавихме `FastCommentsLiveChat`, предварително зададен чат върху същия двигател, който отразява чата на нашия Android SDK: хронологични съобщения с най-новите отдолу, композитор под списъка, жива лента с заглавие с точка за връзка и брой потребители, автоматично превъртане, което паузира докато четете по-стари съобщения, и неограничена история, докато превъртате нагоре. Всеки предварително зададен вариант може да бъде заменен чрез `config`.

```tsx
<FastCommentsLiveChat config={{ tenantId: 'demo', urlId: 'my-room' }}/>
```

### Сега и на уеб

Същият SDK вече работи на уеб чрез `react-native-web`. Богатият текстов композитор (със силата на `react-native-enriched`) рендира по същия начин на iOS, Android и браузъра, така че опитът при редактиране е последователен навсякъде с една единствена имплементация. Накладки, които иначе коментарният списък би изрязал (менюта, избор на GIF, списък с известия) са прикрепени под техните задействия в уеб версията.

### Поддържане на актуалност с React Native

5.0 е изградена и тествана спрямо React Native 0.81 и React 19, и е насочена към Новата архитектура (Fabric), което изисква редактора за богат текст. Поддържането на актуалност тук не е просто работа: редакторът, управлението на жестове и рендирането стават по-бързи и по-точни, докато React Native напредва и предпочитаме да приемем тези подобрения постепенно, отколкото да изостанем с години.

### В заключение

Този пренапис беше за даването на SDK за React Native същия първокласен статус като нашите други библиотеки: бърз и предсказуем модел на данни, външен вид, който можете да темирате с един пропс, уиджет за чат и поддръжка на уеб, всичко това върху модерна основа на React Native.

Можете да намерите SDK на <a href="https://www.npmjs.com/package/fastcomments-react-native-sdk" target="_blank">NPM</a> и източника, с примери, на <a href="https://github.com/FastComments/fastcomments-react-native-sdk" target="_blank">GitHub</a>. Кажете ни долу, ако срещнете нещо.
