### Що нового

Ми щойно випустили версію 5.1 `fastcomments-react-native-sdk`, нашої бібліотеки React Native, яка відображає коментарі та чат з реальними нативними компонентами, а не в WebView.

Ми переписали внутрішню реалізацію, переосмислили весь вигляд, додали віджет живого чату, перенесли SDK на веб та оновили його до останніх версій React Native та React.

### Чому дві бібліотеки для React Native?

Коротке зауваження, оскільки ми отримуємо це запитання. Ми пропонуємо два варіанти:

- `fastcomments-react-native` — це тонка обгортка навколо нашого веб-віджету, що працює в WebView. Це найшвидший спосіб отримати всі функції одразу, і він автоматично отримує виправлення з вебу.
- `fastcomments-react-native-sdk` (цей) відображає UI за допомогою нативних компонентів React Native без WebView. Це більш гнучке рішення, повністю темізоване та відчуває себе нативно, оскільки є нативним.

Для найкращого досвіду ми рекомендуємо SDK. Далі в цій статті розповідається про зміни в 5.0.

### Нове управління станом

Головною метою цієї зміни є забезпечити, щоб наша бібліотека залишалася вірною нашій назві і була швидкою. Ми отримували скарги від кількох клієнтів на продуктивність, тому це тепер виправлено.

Спочатку SDK зберігав своє дерево коментарів у Hookstate. Це працювало, але з ростом потоків і живих оновлень все почало сповільнюватися.

Ми замінили Hookstate на Zustand та плоске індексоване сховище. Коментарі тепер зберігаються в карті `byId` разом з індексами `childrenByParent`, `rootOrder` і `pinnedIds`, замість дерева, яке було вкладене в стан.

Переваги:

- Живі події (новий коментар, голос, редагування) стали O(1) мутаціями замість проходження та клонування дерева.
- Ми відмовилися від двох повноцінних клонирів JSON дерева, які використовувалися при кожному запиті.
- Компоненти підписуються лише на ті частини, які вони читають, за стандартною моделлю селекторів, тому голосування за одним коментарем впливає лише на один коментар.

Цей останній пункт важливіший, ніж може здатися. Завдяки підпискам на основі селекторів, рядок повторно візуалізується тільки коли змінюються його власні дані.

### Редизайн на основі токенів

Старий вигляд був набором закодованих стилів. Новий за замовчуванням генерується з набору семантичних дизайн-токенів (`FastCommentsTheme`): кольори, відступи, радіуси, розміри шрифтів, жорсткість шрифтів і розміри аватарів. Всі стилі походять з цих токенів.

Це означає, что перегрупування стилів – це один пропс:

```tsx
<FastCommentsLiveCommenting config={config} theme={{ colors: { primary: '#FF5500' } }}/>
```

Темний режим – за один набір токенів:

```tsx
import { getDarkTheme } from 'fastcomments-react-native-sdk';

<FastCommentsLiveCommenting config={config} theme={getDarkTheme()}/>
```

Сам редизайн має більш чистий, сучасний нейтральний вигляд: тонкі розділювачі, кнопки голосування в формі капсул та чіпси, заповнені основні кнопки, округлені аватари і послідовна шкала шрифтів. Пропс `styles` все ще доступний для точкових налаштувань, щоб існуючі інтеграції продовжували працювати.

### Спеціальний віджет живого чату

Ми додали `FastCommentsLiveChat`, чат-пресет на тому ж рушії, який відображає чат-погляд нашого Android SDK: хронологічні повідомлення з найновішими внизу, композитор під списком, стрічка живого заголовка з індикатором підключення та кількістю користувачів, автоматичний скролл, який зупиняється, поки ви читаєте старі повідомлення, та безкінечна історія при прокрутці вгору. Кожен пресет може бути переопрацьований через `config`.

```tsx
<FastCommentsLiveChat config={{ tenantId: 'demo', urlId: 'my-room' }}/>
```

### Тепер також на вебі

Той самий SDK тепер працює на вебі через `react-native-web`. Редактор збагаченого тексту (на основі `react-native-enriched`) відображається однаково на iOS, Android та в браузері, тому досвід редагування є однаковим скрізь з єдиною реалізацією. Накладки, які зазвичай перекриваються списком коментарів (меню, вибір GIF, список сповіщень), закріплені під своїми тригерами в веб-версії.

### Підтримка актуальності з React Native

5.0 побудований та протестований на React Native 0.81 і React 19, і він націлений на Нову Архітектуру (Fabric), яка вимагається для нативного редактора збагаченого тексту. Продовження актуальності в цій області не є простою рутиною: редактор, обробка жестів і рендеринг стають швидшими та правильнішими в міру розвитку React Native, а ми б за краще отримувати ці оновлення поступово, ніж відставати на роки.
