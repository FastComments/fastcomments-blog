# [postlink]Захист коментарів за допомогою єдиного входу (SSO)[/postlink]

{{#unless isPost}}
Не вважаєте свої коментарі публічними? Маєте налаштований SSO для авторизації? Це може бути для вас.
{{/unless}}

{{#isPost}}

### <i class="circle">!</i> Ця стаття містить технічну термінологію

#### Вступ

FastComments SSO <a href="/(4-13-2020)-setting-up-sso-with-fastcomments.html" target="_blank">(докладніше тут)</a> надає вашим користувачам можливість
коментувати без необхідності входити в іншу платформу.

Однак лише цього недостатньо для захисту ваших коментарів, оскільки за замовчуванням дані коментарів є публічно доступною інформацією - будь-хто, хто може переглядати
сторінку, може бачити коментарі. Однак це оновлення змінює цю ситуацію.

#### Налаштування без коду

Ми можемо запобігти перегляду та взаємодії з нашими коментарями, коли SSO налаштовано, створивши <a href="https://fastcomments.com/auth/my-account/customize-widget" target="_blank">правило налаштування</a>.

При цьому знайдіть SSO, і ви знайдете цю опцію:

<div class="text-center">
    <img src="/images/fc-sso-protection.png" title="Опція налаштування для захисту коментарів через SSO" alt="Опція налаштування для захисту коментарів через SSO" class="lozad" />
</div>

Увімкніть її та збережіть правило налаштування.

#### Захист лише певного домену або сторінки

Щоб захистити лише певний домен або сторінку, ми просто налаштуємо правило налаштування.

У верхній частині інтерфейсу налаштування ми знайдемо два поля введення: Домен та URL ID.

<div class="text-center">
    <img src="/images/fc-sso-protection-by-page.png" title="Опція налаштування за сторінкою або доменом" alt="Опція налаштування за сторінкою або доменом" class="lozad" />
</div>

Щоб захистити певний домен, введіть домен у відповідне поле "домен".

Щоб захистити певну сторінку, введіть URL сторінки у поле "URL ID". Якщо у вас є користувацька інтеграція з FastComments, ви можете ввести тут тип ID
замість URL.

#### Захист за межами читання

Увімкнення цієї опції захистить сторінку або домен від коментування, якщо користувач не увійшов через SSO. 

#### Усі нюанси

Будь-які користувачі, які створили коментарі до вашої інтеграції SSO, не зможуть їх побачити, якщо не увійдуть через вашу інтеграцію SSO.

#### На завершення

Сподіваємося, що цей посібник виявився для вас корисним та легким для розуміння. Не соромтеся залишити коментарі нижче з будь-якими питаннями.

На все добре!
