### Neuigkeiten

FastComments hat es schon immer ermöglicht, Kommentarstränge über Code oder Anpassungsregeln auf schreibgeschützt zu setzen, aber jetzt kann dies ganz einfach direkt im Kommentar-Widget erfolgen.

Im oberen Drei-Punkte-Menü des Kommentarbereiches haben Administratoren oder Personen mit Moderationsberechtigung jetzt die Möglichkeit, einen Kommentarstrang **zu schließen** und ihn später jederzeit **wieder zu öffnen**.

Dies unterscheidet sich leicht von der Einstellung des Strangs auf `readonly`, die Sie möglicherweise bisher verwendet haben. Einen Strang auf `readonly` zu setzen bedeutet, dass sich absolut nichts auf der Seite ändern kann - keine neuen Kommentare, keine Abstimmungen, keine Bearbeitung von Kommentaren, keine Löschung von Kommentaren.

Einen Kommentarstrang zu schließen ist etwas anders - Kommentare können nicht hinzugefügt werden, aber Benutzer können ihre Kommentare weiterhin löschen, wenn sie dies wünschen. Zusätzlich können weiterhin Abstimmungen abgegeben oder entfernt werden.

### Live

Wie der Rest von FastComments ist das Schließen und Wiederöffnen von Kommentarsträngen live und beeinflusst sofort alle Benutzer, die diesen Strang ansehen.

### So erhalten Sie es

Dieses Feature wurde an alle Kunden ausgerollt!

### Textanpassungen

Eine neue Übersetzung, `THREAD_CLOSED`, mit dem Standardwert auf Englisch `Dieser Kommentarstrang wurde für neue Kommentare geschlossen.` wurde hinzugefügt.

Dieser Text kann immer über die Benutzeroberfläche zur Widget-Anpassung angepasst werden.

### Fazit

Wie bei allen wichtigen Veröffentlichungen freuen wir uns, dass wir uns die Zeit nehmen konnten, dieses Feature zu optimieren, zu testen und richtig zu veröffentlichen. Lassen Sie uns unten wissen, wenn Sie Probleme entdecken.

Prost!
