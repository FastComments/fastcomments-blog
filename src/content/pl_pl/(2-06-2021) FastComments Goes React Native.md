### <i class="circle">!</i> Ten artykuł zawiera terminologię techniczną

### Co nowego

FastComments ma teraz komponenty React Native, które są równorzędne z naszym widgetem VanillaJS.

Postanowiliśmy nazwać bibliotekę `fastcomments-react-native`. Możesz znaleźć repozytorium na GitHubie z kodem źródłowym <a href="https://github.com/FastComments/fastcomments-react-native" target="_blank">tutaj</a>.

Repozytorium zawiera również przykład użycia widgetu.

Jest dostępne na NPM, które możesz znaleźć <a href="https://www.npmjs.com/package/fastcomments-react-native" target="_blank">tutaj</a>.

### Jak to działa

Nowy komponent działa tak, że komponent React jest wrapperem wokół istniejącego widgetu FastComments.

Oznacza to, że jeśli dodamy nowe funkcje do naszych istniejących komponentów lub naprawimy błędy, od razu skorzystasz z tych korzyści!

### Czas rozwoju

Widgety React Native można dodać do swojej aplikacji w zaledwie kilka minut przy minimalnym czasie potrzebnym na rozwój.

### SSO

Wszystkie biblioteki React Native wspierają SSO natywnie, a proces integracji SSO jest taki sam jak w komponentach Vanilla JS. Istniejące
przykłady kodu backendowego również mają zastosowanie.

### Niestandardowe stylizacje?

Komponenty React Native można stylizować za pomocą CSS za pośrednictwem interfejsu użytkownika dostosowywania widgetów, tak jak w przypadku widgetów opartych na sieci.

### Alternatywy

Możesz zbudować własny interfejs użytkownika wokół 

### Na zakończenie

Wydając bibliotekę `fastcomments-react-native` oraz przyszłe biblioteki, mamy nadzieję, że ułatwimy deweloperom przyjęcie FastComments przy użyciu nowoczesnych metodologii rozwoju.

Na zdrowie!
