# [postlink]עדכון SDK למערכת תגובות ב-React Native[/postlink]

{{#unless isPost}}
שחזרנו את ה-fastcomments-react-native-sdk מהיסוד: ניהול מצב חדש ויעיל יותר, עיצוב מחדש באמצעות Design Tokens, ווידג'ט צ'אט חי ייחודי, ותמיכה מעולה באתר.

### <i class="circle">!</i> המאמר הזה מכיל מושגים טכניים

### מה חדש

ממש לאחרונה שחררנו את הגרסה 5.1 של `fastcomments-react-native-sdk`, ספריית ה-React Native שלנו שמציגה תגובות וצ'אט עם רכיבים מקוריים אמיתיים במקום WebView.

שכתבנו את הפנימיות, עיצבנו מחדש את כל המראה, הוספנו ווידג'ט צ'אט חי, הבנו את ה-SDK לאינטרנט, ושדרגנו אותו לגרסה האחרונה של React Native ו-React.

### למה יש שתי ספריות React Native?

הערה מהירה, כי אנחנו מקבלים את השאלה הזו. אנחנו מספקים שתי אפשרויות:

- `fastcomments-react-native` הוא עטיפה דקה סביב ווידג'ט האינטרנט שלנו שרץ ב-WebView. זו הדרך המהירה ביותר לקבל כל תכונה מיידית, והיא נהנית מתיקוני אינטרנט באופן אוטומטי.
- `fastcomments-react-native-sdk` (זה) מציג את הממשק עם רכיבי React Native מקוריים ללא WebView. זה יותר גמיש, ניתן לעצב אותו באופן מלא, ומרגיש מקורי כי הוא מקורי.

לניסיון הטוב ביותר אנו ממליצים על ה-SDK. השאר של פוסט זה עוסק במה השתנה ב-5.0.

### ניהול מצב חדש

המניע העיקרי לשינוי הזה הוא לוודא שהספרייה שלנו נשארת נאמנה לשמה ונשארת מהירה. היו לנו מספר לקוחות שהתלוננו
על הביצועים שלה, אז זה תוקן כעת.

ה-SDK התחיל לשמור את עץ התגובות שלו ב-Hookstate. זה עבד, אבל ככל שכמות התגובות והעדכונים החיוביים גדלה, הדברים התחילו להאט.

החלפנו את Hookstate ב-Zustand וחנות שטוחה עם אינדוקס. תגובות עכשיו חיות במפת `byId` לצד אינדקסים של `childrenByParent`, `rootOrder`, ו-`pinnedIds`, במקום עץ מקונן בתוך מצב.

התוצאה:

- אירועים חיים (תגובה חדשה, הצבעה, עריכה) הפכו להפרות O(1) במקום לעבור ולהעתיק עץ.
- הפסקנו שני העתקות JSON מעמוקות של העץ שלפני כל בקשה.
- רכיבים מתממשקים בדיוק לחלקים שהם קוראים, מודל הבחירה הסטנדרטי, כך שהצבעה על תגובה אחת משפיעה על תגובה אחת.

הנקודה האחרונה חשובה יותר ממה שזה נשמע. עם מנויים מבוססי בחירות, שורה מתחדשת רק כאשר הנתונים שלה משתנים.

### עיצוב מחדש מבוסס Tokens

המראה הישן היה אוסף של סגנונות קשוחים. ברירת המחדל החדשה נוצרת מקבוצת טוקנים עיצוב סמנטיים (`FastCommentsTheme`): צבעים, ריווח, רדיוסים, גדלי גופן, משקלי גופן, וגדלי תמונות פרופיל. כל עץ הסגנון נגזר מאותם טוקנים.

זה אומר שעיצוב מחדש הוא פרופ אחד:

```tsx
<FastCommentsLiveCommenting config={config} theme={{ colors: { primary: '#FF5500' } }}/>
```

מצב כהה הוא רחוק סט טוקנים אחד:

```tsx
import { getDarkTheme } from 'fastcomments-react-native-sdk';

<FastCommentsLiveCommenting config={config} theme={getDarkTheme()}/>
```

העיצוב מחדש עצמו הוא מראה נקי ומודרני: מפרידים דקים, כפתורי הצבעה וסיכות עגולות, כפתורי ראשיים מלאים, תמונות פרופיל מעוגלות, וסולם טיפוגרפי עקבי. הפרופ `styles` עדיין קיים עבור שינויים נקודתיים, כך ששילובים קיימים ממשיכים לעבוד.

### ווידג'ט צ'אט חי ייחודי

הוספנו את `FastCommentsLiveChat`, ווידג'ט צ'אט על אותו מנוע שמחזיר את תצוגת הצ'אט של ה-SDK שלנו לאנדרואיד: הודעות כרונולוגיות עם החדשות ביותר בתחתית, מחבר למטה ברשימה, רצועת כותרת חיה עם נקודת חיבור ומניין משתמשים, גלילה אוטומטית שנעצרת בזמן שאתה קורא הודעות ישנות, והיסטוריה אינסופית ככל שגללים למעלה. כל ווידג'ט ניתן לשינוי דרך `config`.

```tsx
<FastCommentsLiveChat config={{ tenantId: 'demo', urlId: 'my-room' }}/>
```

### עכשיו גם באינטרנט

אותו SDK עכשיו רץ באינטרנט דרך `react-native-web`. מחבר הטקסט המפואר (מופעל על ידי `react-native-enriched`) מציג את אותו הדבר ב-iOS, אנדרואיד, ודפדפן, כך שהניסיון בעריכה עקבי בכל מקום עם יישום אחד. שכבות שברשימת התגובות היו מכסות (תפריטים, בוחר GIF, רשימת התראות) ממוקמות מתחת לגורמים המעוררים שלהן בגרסת האינטרנט.

### להישאר עדכני עם React Native

5.0 נבנתה ונבדקה נגד React Native 0.81 ו-React 19, ומיועדת לארכיטקטורה החדשה (Fabric), אותה עורך טקסט עשיר מקורי דורש. להישאר עדכני כאן אינו רק עבודה לחינם: העורך, טיפול במחוות, וההצגה הופכים למהירים ויותר נכונים ככל ש-React Native מתקדם, והיינו מעדיפים לקחת את השדרוגים האלה בהדרגה מאשר להישאר מאחור שנים.

### לסיכום

השכתוב הזה היה על מתן ה-SDK של React Native את אותה רמת איכות כמו לספריות האחרות שלנו: מודל נתונים מהיר וניתן לצפייה, מראה שניתן לעצב בפרופ אחד, ווידג'ט צ'אט, ותמיכה באינטרנט, הכל על בסיס מודרני של React Native.
