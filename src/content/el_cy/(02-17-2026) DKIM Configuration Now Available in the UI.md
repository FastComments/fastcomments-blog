# [postlink]Η Διαμόρφωση DKIM Είναι Τώρα Διαθέσιμη στη Διεπαφή Χρήστη[/postlink]

{{#unless isPost}}
Η FastComments τώρα παρέχει μια διεπαφή χρήστη για τη διαμόρφωση του DKIM, διευκολύνοντας τη βελτίωση των ποσοστών παράδοσης email χωρίς να χρησιμοποιείτε το API.
{{/unless}}

{{#isPost}}

### Τι Νέο Υπάρχει

Η ρύθμιση του DKIM για το προσαρμοσμένο email domain σας έγινε πιο εύκολη. Προηγουμένως, η διαμόρφωση του DKIM απαιτούσε τη χρήση του
<a href="https://docs.fastcomments.com/guide-api.html#domain-config-structure" target="_blank">Domain Configs API</a> απευθείας. Τώρα μπορείτε να
ρυθμίσετε το DKIM απευθείας από τη διεπαφή χρήστη της FastComments.

### Γιατί Είναι Σημαντικό το DKIM

Όταν προσαρμόζετε το From Domain για τα emails που στέλνει η FastComments εκ μέρους σας, οι πάροχοι email χρειάζεται να επαληθεύσουν ότι
αυτά τα emails είναι νόμιμα. Το DKIM (DomainKeys Identified Mail) είναι ένας από τους βασικούς μηχανισμούς αυθεντικοποίησης που βοηθά σε αυτό.
Χωρίς σωστή διαμόρφωση DKIM, τα emails που στέλνονται εκ μέρους του τομέα σας είναι πιο πιθανό να καταλήξουν σε φακέλους ανεπιθύμητης αλληλογραφίας.

### Πώς Να Το Ρυθμίσετε

Μεταβείτε στη σελίδα <a href="https://fastcomments.com/auth/my-account/configure-domains" target="_blank">Τα Τομέα Μου</a> και κάντε κλικ
"Εμφάνιση Προχωρημένων" για τον τομέα που θέλετε να ρυθμίσετε. Θα δείτε τις ρυθμίσεις DKIM όπου μπορείτε να εισαγάγετε το όνομα του τομέα σας, τον επιλεγέα κλειδιών,
και το ιδιωτικό κλειδί.

Μετά την αποθήκευση, η διαμόρφωση DKIM θα εφαρμοστεί σε όλα τα emails που αποστέλλονται από αυτόν τον τομέα.

### Το API Λειτουργεί Ακόμη

Εάν προτιμάτε να διαχειρίζεστε τη διαμόρφωση DKIM προγραμματιστικά, το
<a href="https://docs.fastcomments.com/guide-api.html#domain-config-structure" target="_blank">Domain Configs API</a> συνεχίζει να λειτουργεί
όπως πριν.

### Στο Συμπέρασμα

Ελπίζουμε αυτό να διευκολύνει τη διασφάλιση ότι οι χρήστες σας θα λάβουν τα emails που στέλνει η FastComments εκ μέρους σας. Ενημερώστε μας
κάτω αν έχετε οποιαδήποτε ερώτηση.

Cheers!

{{/isPost}}
