### Τι είναι Νέο

Μόλις κυκλοφόρησε η έκδοση 5.1 του `fastcomments-react-native-sdk`, της βιβλιοθήκης μας για React Native που αποδίδει σχόλια και συνομιλία με πραγματικά εγγενή στοιχεία αντί για μια WebView.

Επαναγράψαμε τα εσωτερικά της, επανασχεδιάσαμε την όλη εμφάνιση, προσθέσαμε ένα widget ζωντανής συνομιλίας, μεταφέραμε το SDK στον ιστό, και το αναβαθμίσαμε στην πιο πρόσφατη έκδοση του React Native και του React.

### Γιατί Δύο Βιβλιοθήκες React Native;

Μια γρήγορη σημείωση, καθώς έχουμε αυτή την ερώτηση. Έχουμε δύο επιλογές:

- `fastcomments-react-native` είναι μια λεπτή επικάλυψη γύρω από το web widget μας που τρέχει σε μια WebView. Είναι ο ταχύτερος τρόπος για να αποκτήσετε κάθε δυνατότητα αμέσως, και επωφελείται αυτόματα από διορθώσεις στον ιστό.
- `fastcomments-react-native-sdk` (αυτό) αποδίδει την UI με εγγενή στοιχεία του React Native χωρίς WebView. Είναι πιο ευέλικτο, πλήρως θεματικό, και φαίνεται εγγενές επειδή είναι εγγενές.

Για την καλύτερη εμπειρία προτείνουμε το SDK. Το υπόλοιπο αυτού του άρθρου αφορά τις αλλαγές στην έκδοση 5.0.

### Νέα Διαχείριση Κατάστασης

Ο κύριος λόγος αυτής της αλλαγής είναι να διασφαλίσουμε ότι η βιβλιοθήκη μας παραμένει πιστή στο όνομά της και παραμένει γρήγορη. Είχαμε αρκετούς πελάτες να παραπονιούνται για την απόδοση της, οπότε αυτό διορθώθηκε τώρα.

Αρχικά, το SDK κρατούσε το δέντρο σχολίων του σε Hookstate. Λειτουργούσε, αλλά καθώς οι threads και οι ζωντανές ενημερώσεις αυξάνονταν, τα πράγματα άρχισαν να επιβραδύνονται.

Αντικαταστήσαμε το Hookstate με το Zustand και μια επίπεδη, ευρετηριασμένη αποθήκη. Τα σχόλια τώρα βρίσκονται σε έναν χάρτη `byId` μαζί με τα ευρετήρια `childrenByParent`, `rootOrder`, και `pinnedIds`, αντί για ένα δέντρο μέσα σε κατάσταση.

Το αποτέλεσμα:

- Οι ζωντανές εκδηλώσεις (ένα νέο σχόλιο, μια ψήφος, μια επεξεργασία) έγιναν O(1) μετατροπές αντί να πρέπει να περπατούν και να κλωνοποιούν ένα δέντρο.
- Ξεφορτωθήκαμε δύο πλήρη δέντρα JSON deep-clones που τρέχαμε σε κάθε fetch.
- Τα στοιχεία subscribe ακριβώς στα κομμάτια που διαβάζουν, το τυπικό μοντέλο επιλεγμένης, έτσι μια ψήφος σε ένα σχόλιο επηρεάζει μόνο εκείνο το σχόλιο.

Αυτό το τελευταίο σημείο έχει περισσότερη σημασία από ό,τι ακούγεται. Με τις επιλεγμένες συνδρομές, μια γραμμή ξανασχεδιάζεται μόνο όταν αλλάζουν τα δικά της δεδομένα.

### Μια Επανασχεδίαση Με Βάση Tokens

Η παλιά εμφάνιση ήταν σωρός σκληρά κωδικοποιημένων στυλ. Η νέα προεπιλογή παράγεται από ένα σύνολο σημασιολογικών σχεδιαστικών tokens (`FastCommentsTheme`): χρώματα, αποστάσεις, ακτίνες, μεγέθη γραμματοσειρών, βάρη γραμματοσειρών, και μεγέθη avatar. Ολόκληρο το δέντρο στυλ προέρχεται από αυτά τα tokens.

Αυτό σημαίνει ότι η επανασχεδίαση είναι μια ιδιότητα:

```tsx
<FastCommentsLiveCommenting config={config} theme={{ colors: { primary: '#FF5500' } }}/>
```

Η νυχτερινή λειτουργία είναι ένα σύνολο tokens μακριά:

```tsx
import { getDarkTheme } from 'fastcomments-react-native-sdk';

<FastCommentsLiveCommenting config={config} theme={getDarkTheme()}/>
```

Η επανασχεδίαση είναι καθαρή, σύγχρονη ουδέτερη εμφάνιση: λεπτές διαχωρίσεις, κουμπιά ψήφου και chips σε μορφή χάπι, γεμάτα κύρια κουμπιά, στρογγυλεμένα avatars, και συνεκτική κλίμακα τύπου. Η ιδιότητα `styles` είναι ακόμα εκεί για χειρουργικές παρακάμψεις, ώστε οι υπάρχουσες ενσωματώσεις να συνεχίζουν να λειτουργούν.

### Ένα Αφιερωμένο Widget Ζωντανής Συνομιλίας

Προσθέσαμε το `FastCommentsLiveChat`, μια προεπιλεγμένη συνομιλία πάνω από την ίδια μηχανή που αντικατοπτρίζει την προβολή συνομιλίας του Android SDK μας: χρονολογημένα μηνύματα με το πιο πρόσφατο στο κάτω μέρος, τον συνθέτη κάτω από τη λίστα, μια ζωντανή ταινία κεφαλίδας με μια κουκίδα σύνδεσης και αριθμό χρηστών, αυτόματη κύλιση που σταματάει ενώ διαβάζετε παλιότερα μηνύματα, και άπειρη ιστορία καθώς κάνετε κύλιση προς τα πάνω. Κάθε προεπιλεγμένη ρύθμιση είναι ανατρέψιμη μέσω του `config`.

```tsx
<FastCommentsLiveChat config={{ tenantId: 'demo', urlId: 'my-room' }}/>
```

### Τώρα και στον Ιστό

Το ίδιο SDK τρέχει τώρα στον ιστό μέσω του `react-native-web`. Ο πλούσιος σύνθετης κειμένου (καθοδηγούμενος από το `react-native-enriched`) αποδίδει με τον ίδιο τρόπο σε iOS, Android, και τον φυλλομετρητή, οπότε η εμπειρία επεξεργασίας είναι συνεπής παντού με μία εφαρμογή. Οι επικαλύψεις που η λίστα σχολίων θα επηρεάζει διαφορετικά (μενού, το picker GIF, η λίστα ειδοποιήσεων) είναι προσκολλημένες κάτω από τους θορύβους τους στην έκδοση ιστού.

### Διατηρώντας Ενημερωμένο με το React Native

Η 5.0 έχει κατασκευαστεί και δοκιμαστεί σε React Native 0.81 και React 19, και στοχεύει στη Νέα Αρχιτεκτονική (Fabric), η οποία απαιτεί ο εγγενής επεξεργαστής πλούσιου κειμένου. Η διατήρηση εδώ δεν είναι δουλειά χωρίς νόημα: ο επεξεργαστής, η διαχείριση κινήσεων, και η απόδοση γίνονται όλα πιο γρήγορα και πιο σωστά καθώς το React Native προχωρά, και θα προτιμούσαμε να παίρνουμε αυτές τις αναβαθμίσεις σταθερά παρά να μείνουμε πίσω για χρόνια.
