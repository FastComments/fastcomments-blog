# [postlink]Ανακοίνωση Αρχείου Ελέγχου[/postlink]

{{#unless isPost}}Η FastComments τώρα παρέχει ένα λεπτομερή λογιστικό αρχείο όλων των τροποποιήσεων που έγιναν μέσω της πλατφόρμας.{{/unless}}

{{#isPost}}

### Τι είναι νέο

Από τον Μάρτιο, η FastComments συμπεριλαμβάνει λεπτομερή έλεγχο με το σχέδιο Pro και για όλους τους υπάρχοντες πελάτες εταιρειών.

Όλες οι διοικητικές ενέργειες, είτε πρόκειται για πρόσβαση σε δεδομένα, τροποποίηση ενός σχολίου ή αποδέσμευση ενός χρήστη από αποκλεισμό, παρακολουθούνται και καταχωρούνται στο λογιστικό αρχείο.

### Πώς να το χρησιμοποιήσετε

Το Αρχείο Ελέγχου μπορεί να προσπελαστεί πηγαίνοντας στο `Διαχείριση Δεδομένων` και στη συνέχεια `Αρχεία Ελέγχου`. Ένας άμεσος σύνδεσμος είναι [εδώ](https://fastcomments.com/auth/my-account/manage-data/audit-logs).

Για να ξεκινήσετε, θα δείτε μια σελίδα όπως η παρακάτω:

<div class="text-center">
    <img src="/images/audit-log-main.png" alt="Audit Log" />
</div>

Από εδώ, μπορείτε να αναζητήσετε με βάση το όνομα χρήστη, τη διεύθυνση IP, τον τύπο ενέργειας ή το πόρο.

Για παράδειγμα, φιλτράροντας με βάση το πόρο:

<div class="text-center">
    <img src="/images/audit-log-resource.png" alt="Audit Log Resource Selection" />
</div>

Μπορούμε επίσης να εξετάσουμε τις λεπτομέρειες μιας καταχώρησης στο λογιστικό αρχείο και ακόμη και να δούμε το σχετικό αντικείμενο που άλλαξε.

Για παράδειγμα, μπορούμε να εξετάσουμε μια καταχώρηση για τον αποκλεισμό ενός χρήστη:

<div class="text-center">
    <img src="/images/audit-log-details.png" alt="Audit Log Entry Details" />
</div>

Εδώ μπορούμε να δούμε ποιος απαγόρευσε τον χρήστη (ένας χρήστης με το όνομα χρήστη `winrid`). Μπορούμε να δούμε ότι είναι μόνιμος αποκλεισμός, και έχουμε επίσης το id του χρήστη και το email που αποκλείστηκε, μαζί με το κείμενο για το οποίο αποκλείστηκε.

Μπορούμε να δούμε τη διεύθυνση IP που δημιούργησε τον αποκλεισμό, αλλά η διεύθυνση IP του αρχικού σχολιαστή παραλείπεται σκόπιμα για την προστασία της ιδιωτικότητάς του.

Για να δούμε όλα τα γεγονότα από τον `winrid`, μπορούμε απλά να κάνουμε κλικ στο εικονίδιο φίλτρου στα δεξιά, δίπλα στο όνομα χρήστη τους.

### Περιορισμός Πρόσβασης

Τα δεδομένα στο λογιστικό αρχείο μπορούν να προσπελαστούν μόνο από χρήστες με άδεια `Διαχείριση Δεδομένων`.

### Διατήρηση

Όλες οι καταχωρήσεις στο λογιστικό αρχείο διατηρούνται για δύο χρόνια. Ωστόσο, το λογιστικό αρχείο καθαρίζεται όταν διαγραφεί ο λογαριασμός σας.

### Πρόσβαση API

[Η τεκμηρίωση για το API του Αρχείου Ελέγχου είναι εδώ.](https://docs.fastcomments.com/guide-api.html#audit-log-structure) Θα διαπιστώσετε ότι το λογιστικό αρχείο μπορεί να 
ταξινομηθεί και να σελιδοποιηθεί με αρκετούς διαφορετικούς τρόπους ανάλογα με την περίπτωση χρήσης σας.

### Υπάρχοντες Πελάτες

Εάν είστε στο σχέδιο Pro ή αντίστοιχο, θα διαπιστώσετε ότι το λογιστικό αρχείο έχει ενεργοποιηθεί αυτόματα! Εάν επιθυμείτε να αποκτήσετε πρόσβαση, παρακαλώ επικοινωνήστε με τον εκπρόσωπό σας υποστήριξης.

### Τι παραλείπεται

Ο μόνος τύπος αλλαγών που παραλείπεται είναι τα πραγματικά σχόλια και οι ψήφοι τους. Αυτό συμβαίνει επειδή έχουμε διαπιστώσει ότι οι πελάτες μας ενδιαφέρονται κυρίως για τη διοικητική πλευρά κατά τον έλεγχο και όχι απαραίτητα για την παρακολούθηση του τι σχολιάζουν οι άνθρωποι. Αυτές οι τύποι ενεργειών προσθέτουν επίσης σημαντικό θόρυβο στο λογιστικό αρχείο.

Ωστόσο, εάν αυτή είναι μια απαίτηση, επικοινωνήστε μαζί μας και μπορούμε να συζητήσουμε την ενεργοποίηση αυτού για εσάς.

### Στο Συμπέρασμα

Ελπίζουμε ότι θα βρείτε αυτόν τον τομέα της προσφοράς μας χρήσιμο, το UI εύκολο στη χρήση και το API γρήγορο. 

Χαιρετισμούς!
